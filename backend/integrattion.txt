# PHASE 1: Integration Guide

This guide walks you through integrating the three new service files into your backend.

## Files to Create/Copy

### 1. Create `backend/services/__init__.py`

```python
# backend/services/__init__.py
from services.environment import VirtualEnvironment, Agent, WorldObject
from services.agent import AgentService
from services.llm import LLMService

__all__ = [
    "VirtualEnvironment",
    "Agent",
    "WorldObject",
    "AgentService",
    "LLMService"
]
```

### 2. Copy the service files

Copy these three files to the specified locations:

**File 1:** `environment.py` → `backend/services/environment.py`
- The game engine that manages world state and executes actions

**File 2:** `agent.py` → `backend/services/agent.py`
- The harness that coordinates environment + LLM

**File 3:** `llm.py` → `backend/services/llm.py`
- Claude API integration with tool use

## Understanding What You've Built

### Architecture Layers

```
┌─────────────────────────────────┐
│  Frontend (React + Three.js)     │  (Phase 2)
├─────────────────────────────────┤
│  FastAPI (Routes & WebSocket)   │  (Phase 2)
├─────────────────────────────────┤
│  Agent Service                  │  ← You are here (harness)
├─────────────────────────────────┤
│  LLM Service                    │  ← Claude integration
├─────────────────────────────────┤
│  Environment Service            │  ← Game engine
├─────────────────────────────────┤
│  Database (PostgreSQL)          │  ← Persistence
└─────────────────────────────────┘
```

### Data Flow During Task Execution

1. **Setup Phase**
   ```
   AgentService.run_task(task_description)
   ↓
   Environment.reset() → Agent at [10, 0, 10]
   ↓
   Database: Create EnvironmentSession
   ```

2. **Perception Phase (each step)**
   ```
   Environment.get_observation(task)
   ↓
   Returns: Observation {
       agent_state: {...},
       visible_objects: [{id, type, position, ...}],
       inventory: [...],
       environment_context: {...}
   }
   ```

3. **Reasoning Phase**
   ```
   LLMService.reason_and_act(prompt, tools, observation)
   ↓
   Call Claude with available tools
   ↓
   Claude responds: {type: "tool_use", name: "move", input: {direction: "forward"}}
   ```

4. **Action Phase**
   ```
   Environment.execute_action("move", {direction: "forward"})
   ↓
   Validate action (in bounds? no collision?)
   ↓
   Execute: Agent position [10, 0, 10] → [10, 0, 11]
   ↓
   Return: ActionResult {success: true, message: "...", new_position: [...]}
   ```

5. **Logging Phase**
   ```
   Database: Store AgentAction {
       session_id: 1,
       action_type: "move",
       success: true,
       agent_position_before: [10, 0, 10],
       agent_position_after: [10, 0, 11],
       step_number: 1
   }
   ```

6. **Repeat** (back to perception) until task complete or max steps

## Common Issues & Solutions

### Issue 1: "ModuleNotFoundError: No module named 'services'"

**Solution:** Make sure you're running from the `backend/` directory:
```bash
cd backend/
python -c "from services.environment import VirtualEnvironment"
```

### Issue 2: Database connection fails

**Solution:** Check your `.env` file:
```bash
# Verify DATABASE_URL is correct
grep DATABASE_URL .env

# Should show something like:
# DATABASE_URL=postgresql+asyncpg://postgres:password@localhost:5432/aiteacher
```

### Issue 3: "ImportError: cannot import name 'settings' from 'config'"

**Solution:** Make sure `config.py` has this line at the end:
```python
# At end of config.py
settings = Settings()
```
