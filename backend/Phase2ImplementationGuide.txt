# PHASE 2: FastAPI Server Setup Guide

## Overview

Phase 2 is about **exposing your harness through HTTP endpoints and WebSockets**.

You'll create:
1. `main.py` - FastAPI app initialization
2. `utils/logger.py` - Structured logging
3. Update service imports

## Step 7: Start a Task

This is the big one! Run an agent task:

```bash
curl -X POST "http://localhost:8000/tasks/start" \
  -H "Content-Type: application/json" \
  -d '{
    "task_description": "Navigate to the red cube and examine it",
    "max_steps": 50
  }'
```

The agent will:
1. Get an observation
2. Send it to Claude
3. Claude decides action
4. Execute action
5. Log to database
6. Repeat until complete

Expected response (will take a few seconds):
```json
{
  "status": "success",
  "task_result": {
    "success": true,
    "status": "success",
    "steps_taken": 12,
    "final_position": [5.0, 0.0, 5.0],
    "inventory": ["Red Cube"],
    "session_id": 1,
    "duration_seconds": 5.23
  }
}
```

## Understanding What You've Built

### Main.py Structure

```
main.py
├── Lifespan management
│   ├── startup_event() - Initialize everything
│   └── shutdown_event() - Clean up
├── Middleware
│   └── CORS - Allow requests from React
├── Health check
│   └── /health - Server status
├── Environment routes
│   ├── POST /world/setup - Add objects
│   ├── GET /world/status - Current state
│   └── GET /world/observation - What agent sees
├── Task routes
│   ├── POST /tasks/start - Run agent
│   ├── GET /tasks/{id} - Task results
│   └── GET /tasks/{id}/actions - Step-by-step logs
├── WebSocket
│   └── /ws/{task_id} - Real-time updates
└── Error handling
    └── Global exception handler
```

### Key Concepts

**Lifespan Context Manager:**
- Everything before `yield` runs on startup
- Everything after runs on shutdown
- Ensures proper initialization and cleanup

**Dependency Injection:**
- Services stored in `app_state`
- Access in routes: `app_state["environment"]`
- Single instance per server

**CORS Middleware:**
- Allows React frontend to make requests
- Configured in settings
- Without it, browser blocks cross-origin requests

**WebSocket:**
- Persistent connection for real-time updates
- Client connects to `/ws/{task_id}`
- Can receive streaming updates as agent works

## Common Errors & Solutions

### Error 1: "Module not found: uvicorn"

Solution:
```bash
pip install uvicorn
```

(It's in requirements.txt, so `pip install -r requirements.txt` should have it)

### Error 2: "Anthropic API key not found"

Solution: Check your `.env` file has `ANTHROPIC_API_KEY`:

```bash
grep ANTHROPIC_API_KEY .env
```

If empty, get a key from https://console.anthropic.com/account/keys

### Error 3: Database connection error

Solution: Verify PostgreSQL is running and connection string is correct:

```bash
grep DATABASE_URL .env
```

Should look like:
```
DATABASE_URL=postgresql+asyncpg://postgres:password@localhost:5432/aiteacher
```

### Error 4: Port 8000 already in use

Solution: Use a different port:

```bash
uvicorn main:app --reload --port 8001
```
